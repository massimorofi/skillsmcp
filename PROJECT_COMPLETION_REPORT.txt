================================================================================
                            RESOURCE URIS
================================================================================

Skill resources are accessible via skill:// URI scheme:

Main Instruction File:
  skill://example-skill/SKILL.md

Manifest (JSON file listing):
  skill://example-skill/_manifest
  
  Returns:
  {
    "skill": "example-skill",
    "files": [
      {"path": "SKILL.md", "size": 1234, "hash": "sha256:..."},
      {"path": "reference.md", "size": 567, "hash": "sha256:..."}
    ]
  }

Supporting Files:
  skill://example-skill/reference.md
  skill://example-skill/examples/sample-config.xml

================================================================================
                         CREATING YOUR OWN SKILL
================================================================================

1. Create a directory with a descriptive name:
   mkdir ~/.claude/skills/my-skill

2. Create SKILL.md with instructions:
   cat > ~/.claude/skills/my-skill/SKILL.md << 'END'
   ---
   description: My custom skill for AI assistants
   ---
   
   # My Skill
   
   Instructions for using my skill...
   END

3. Optionally add supporting files:
   echo "Additional docs..." > ~/.claude/skills/my-skill/reference.md
   mkdir -p ~/.claude/skills/my-skill/examples
   cp config.json ~/.claude/skills/my-skill/examples/

4. If reload=true, skill appears immediately
   Otherwise, restart the server:
   ./stop.sh && ./start.sh

5. Test with client:
   python client_example.py list
   python client_example.py read my-skill
